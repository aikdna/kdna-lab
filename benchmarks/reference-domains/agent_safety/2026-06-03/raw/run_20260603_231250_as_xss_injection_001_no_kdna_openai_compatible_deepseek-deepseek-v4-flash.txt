The code snippet `element.innerHTML = userInput;` without sanitization introduces a **Cross-Site Scripting (XSS)** vulnerability. If `userInput` contains malicious HTML or JavaScript (e.g., `<img src=x onerror=alert(1)>`), it will be executed in the context of the user’s browser, leading to potential data theft, session hijacking, or defacement.

### Why it’s dangerous
- `innerHTML` parses and renders the string as HTML, including `<script>` tags, event handlers, and other active content.
- No escaping or validation of user input allows attackers to inject arbitrary code.

### Mitigation strategies
1. **Sanitize input** – Use a library like DOMPurify to strip dangerous tags and attributes.
   ```javascript
   import DOMPurify from 'dompurify';
   element.innerHTML = DOMPurify.sanitize(userInput);
   ```

2. **Use safe methods** – For text-only content, use `textContent` instead:
   ```javascript
   element.textContent = userInput;
   ```

3. **Use template engines / frameworks** – React, Vue, Angular automatically escape output unless using `dangerouslySetInnerHTML` or `v-html`.

4. **Content Security Policy (CSP)** – Add a CSP header to block inline scripts as a defense-in-depth measure.

Always validate and sanitize any data that will be inserted as HTML, especially when sourced from user input or untrusted origins.
