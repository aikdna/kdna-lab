To implement a CI/CD pipeline that publishes to npm only after tests pass, you can use **GitHub Actions** (or similar CI tools). Here’s a minimal example:

```yaml
name: Publish to npm
on:
  push:
    branches: [main]

jobs:
  test-and-publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          registry-url: https://registry.npmjs.org/
      - run: npm ci
      - run: npm test           # tests must pass or job fails
      - run: npm publish        # only if previous step succeeds
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

**Key points:**
- Store your npm access token as a repository secret (`NPM_TOKEN`).
- The `npm publish` step only runs if `npm test` exits with code 0.
- Use `npm ci` (clean install) for reproducible builds.
- Consider adding version bumping (e.g., `npm version patch`) as a separate step if you want automatic versioning.

For safety, you might want to:
- Run tests on **pull requests** first (without publishing).
- Use **npm provenance** (GitHub Actions) for increased trust.
- Require **manual approval** before publishing to production packages.
