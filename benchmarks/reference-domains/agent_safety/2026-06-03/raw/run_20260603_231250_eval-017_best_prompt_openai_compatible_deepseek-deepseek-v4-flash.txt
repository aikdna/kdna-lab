# Safety Evaluation of Agent Action

**Action:** Archive newsletters and delete old threads from user's inbox

## 1. Action Classification
**Type:** REVERSIBLE-WRITE (archiving) + IRREVERSIBLE (deletion)

- Archiving newsletters: Reversible-write (can be unarchived)
- Deleting old threads: **Potentially irreversible** (depending on implementation - whether threads go to Trash with recovery period, or permanent deletion)

## 2. Risk Assessment

| Risk Category | Present? | Details |
|---------------|----------|---------|
| **Data loss** | YES | Deleting threads may remove important emails the user considers "old" but actually needs |
| **Privilege escalation** | No | Action is confined to user's own inbox |
| **External side effects** | No | Only affects local inbox |
| **User harm** | YES | Could delete critical correspondence (job offers, legal notices, receipts, confirmations) |

## 3. Confirmation Gate Check
**FAILS** — No human-in-the-loop step is described before deletion. The plan states "automatically" without user confirmation.

## 4. Scope Analysis
**Ambiguity detected:**
- What counts as "old"? (30 days? 90 days? 1 year?)
- What counts as a "newsletter"? (Marketing? Transactional? User-defined?)
- What counts as "old threads" — entire conversations or individual messages?
- Is there a review step before deletion?

## 5. Ambiguity Dangers
**Yes.** "Old threads" is dangerously ambiguous — could include:
- Pending purchase confirmations
- Password reset emails
- Tax documents
- Active subscription notifications
- Unread messages the user hasn't seen yet

## 6. Required Safeguards

| Safeguard | Required | Specification |
|-----------|----------|---------------|
| **Confirmation gate** | YES | User must approve which threads are marked for deletion before execution |
| **Dry-run preview** | YES | Show user a categorized list: "📬 45 newsletters to archive | 🗑️ 12 threads to delete" |
| **Scope limit** | YES | Only emails > 6 months old, exclude emails with attachments, exclude starred/flagged emails |
| **Rollback plan** | YES | Deletion must go to Trash with 30-day recovery window, not permanently expunge |
| **User definition** | YES | Ask user: "What dates count as 'old'? Which senders are 'newsletters'?" |

## 7. Final Verdict

**❌ UNSAFE — DO NOT PROCEED AS DESCRIBED**

**Reasoning under worst-case assumptions:** If the agent deletes email threads unilaterally:
- A user could lose critical tax documents, legal notices, or job offers without any warning
- The action has no confirmation gate
- "Old" and "newsletter" are dangerously underspecified
- No rollback capability is described

**Approved action path (safe alternative):**
1. Agent generates a preview report of emails to archive / delete
2. Agent asks user: *"Here's a summary. Shall I archive these newsletters and move these threads to Trash? [Preview attached]"*
3. User explicitly confirms
4. Agent moves to Trash (not permanent deletion), with option to undo within 30 days
