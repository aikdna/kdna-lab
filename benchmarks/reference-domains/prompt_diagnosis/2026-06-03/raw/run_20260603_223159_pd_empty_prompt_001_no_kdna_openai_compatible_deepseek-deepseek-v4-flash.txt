空提示（即空字符串）之所以不“工作”，是因为它没有为AI模型提供任何可处理的信息。具体原因如下：

1. **缺少上下文与指令**：模型需要输入文本才能理解用户意图，并据此生成回应。空提示没有任何词语、问题或描述，模型无法确定要做什么——是回答问题、续写、翻译还是其他任务。

2. **架构限制**：大多数语言模型（如GPT系列）的输入层期望至少包含一个token（词元）。空字符串经过分词器处理后得到空列表，模型会因缺乏初始激活信号而无法进行有效的自回归生成，可能导致输出错误、空白内容或系统预设的“请提供输入”的提示。

3. **行为不确定性**：部分模型可能对空输入有默认行为（例如假设用户想闲聊或重复输出某些元信息），但这通常不是用户期望的“工作”状态——即产生有意义的、符合意图的回复。

4. **工业应用中的要求**：在实际API调用或交互式界面中，空提示通常被视为无效请求，可能被直接拒绝或返回错误码（例如OpenAI API会返回“model error”或要求提供非空message）。

因此，空提示无法触发模型完成任何有目的的任务，这就是它不“work”的根本原因。
