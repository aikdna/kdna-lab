## 诊断分析报告

### 原始 prompt
> “帮我写点关于AI的东西。”

---

### 1. 实际意图 vs 隐含假设
- **显式意图**：请求生成一段关于“人工智能”的文本。
- **隐含假设**：用户假设 AI 能自动理解“写点”的具体含义（长度、风格、用途）、知道“AI”应该聚焦哪个子领域（技术原理、伦理影响、行业应用、历史发展等），并默认输出格式为自然段落。
- **问题**：假设过度，导致模型无法确定最优响应。

### 2. 任务模糊性（多重目标？成功标准？）
- **混合目标**：没有明确单一目标。可能同时期望介绍、分析、举例或观点表达，但未指定。
- **成功标准缺失**：无法判断输出是否“好”——是信息准确、通俗易懂、有深度还是有趣？缺少任何可衡量的指标。
- **严重性**：功能性。输出可能随机、肤浅或偏离用户真实需求。

### 3. 缺失上下文
需要补充的关键信息：
- **目标受众**（小学生？程序员？高管？）
- **输出用途**（报告、创意故事、学习笔记、社交媒体帖子？）
- **内容深度**（入门概述、技术细节、批判性分析？）
- **篇幅约束**（一句话？1000字？）
- **风格偏好**（正式、轻松、学术、俏皮？）
- **时间或地区焦点**（最新进展？经典理论？）

### 4. 输出格式期望
- **默认假设**：普通段落文本。但用户可能想要列表、对比表格、代码示例、问答形式或分块大纲。格式不明确导致模型可能选错结构。
- **严重性**：功能性。输出可能不符合用户期望的呈现方式。

### 5. 风险措辞分析
- **“写点”**：过于随意，容易让模型产生“可以随便写”的倾向，降低质量控制意识。
- **“关于AI”**：范围过大，模型可能选择最常见的泛泛内容（如定义），忽略用户潜在专属需求。
- **无否定、无诱导**：无明显安全风险，但存在“内容失控”风险（如模型输出不相关或过于专业/幼稚）。
- **严重性**：功能性问题（非安全）。

### 6. 具体重写建议（修复每个缺陷）

**修复版 prompt**：
> “请写一篇**500字左右的科普短文**，面向**普通成年人**，介绍**人工智能的三个核心应用领域**（如医疗、自动驾驶、自然语言处理），并**用具体例子说明每个领域的实际进展**。语言要求**通俗易懂、避免术语堆砌**。输出格式为**分段正文，每段开头加小标题**。”

**各修复点说明**：
- **“500字左右”** → 明确篇幅，避免过长或过短。
- **“科普短文”“普通成年人”** → 界定受众与深度，防止过于专业或幼稚。
- **“三个核心应用领域”+举例** → 限定内容范围，消除搜索空间的无限性。
- **“通俗易懂、避免术语”** → 指定风格，减少抽象表述。
- **“分段小标题”** → 指定输出格式，提升可读性。

### 7. 严重性评级汇总

| 问题类别 | 严重性 | 原因 |
|----------|--------|------|
| 任务模糊性（多重目标） | 功能性 | 导致模型输出多样性，无法满足用户隐含需求 |
| 缺失上下文（受众/用途/深度） | 功能性 | 模型被迫猜测，产出很可能不匹配 |
| 输出格式未指定 | 功能性 | 可能得到不期望的格式（如列表而非段落） |
| “写点”措辞过于宽泛 | 功能性 | 降低模型对质量的约束 |
| 范围“关于AI”过大 | 功能性 | 内容可能偏离用户真正关心的子主题 |
| 安全风险 | 无 | 无敏感或危险指令 |

**总结**：该 prompt 的主要缺陷是 **“过于开放”**，导致模型无法聚焦。通过增加篇幅、受众、内容焦点、风格和格式约束，可大幅提升输出质量。
