## 诊断报告

### 1. 识别实际请求与隐含假设
- **明确请求**：用户要求AI根据“上季度的数据”制定“下一季度的增长策略”。
- **隐含假设**：
  - AI能够获取或已拥有上季度的数据（但prompt中未提供任何数据）。
  - AI理解“增长”的具体含义（营收、用户数、利润、市场份额？）。
  - AI知道用户所在行业、公司规模、资源状况。
  - “策略”的输出形式（文字报告、列表、表格？）是默认可接受的。

### 2. 任务模糊性
- **多目标混合**：虽然只提了一个总体目标，但“增长策略”本身包含多个子目标（如渠道选择、产品优化、定价调整等），没有指明优先级或约束。
- **成功标准不明确**：什么样的策略算是“好”？需要可量化（如增长20%）还是定性方向？需要可执行方案还是宏观框架？

### 3. 缺失上下文
- **核心数据缺失**：没有提供任何上季度的数据（数字、趋势、图表、文本摘录等），AI无法基于数据做分析。
- **背景信息缺失**：
  - 行业（电商、SaaS、制造业等）
  - 目标用户画像
  - 当前市场环境（竞争、政策、季节因素）
  - 可用资源（预算、人力、技术能力）
  - 时间范围（季度内是否分月？）
- **约束条件缺失**：是否有成本上限、风险偏好、合规要求？

### 4. 输出格式期望
- **未指定格式**：用户没有说明期望的呈现形式。AI可能输出段落文字、清单、表格、思维导图等，而用户可能只想要bullet points或一份完整报告。
- **隐含假设**：默认文本输出，但若数据量大，结构化格式（如表格+建议）会更合适。

### 5. 风险措辞分析
- **过宽泛的权限**：“制定增长策略”给了AI极大的自由度，可能导致生成通用、空洞的建议（例如“加强社交媒体营销”），而不是基于具体数据的定制方案。
- **无否定或引导性问题**，但存在**隐性依赖**：用户假设AI知道“上季度的数据”是什么，但实际prompt中并未给出，这属于**隐含缺失**，而非语言风险。
- **无安全风险**：未涉及敏感内容或危险指令。

### 6. 具体重写与修复说明

**原始prompt**：
> “根据上季度的数据，为我制定下一季度的增长策略。”

**重写示例**：
```
请根据以下上季度数据（见附件/表格）为我的B2B SaaS公司（产品：项目管理工具）制定下一季度的增长策略。  
目标：将新客户月获客数从当前200提升30%。  
限制：营销预算不超过上一季度的110%，团队规模不变。  
请输出一份包含以下部分的策略文档：  
1. 基于数据的核心洞察（列出3个关键发现）  
2. 具体增长举措（每项注明预期影响、所需资源、风险）  
3. 实施时间线（按周划分里程碑）  
4. 关键指标（KPIs）跟踪建议  
格式：Markdown，每个部分用标题隔开。
```

**各修复项说明**：
- **提供数据或明确数据来源**（如“见附件”）→ 解决核心数据缺失，使AI能真正“根据数据”分析。
- **补充行业与产品背景**（B2B SaaS、项目管理工具）→ 提供上下文，避免生成与行业无关的通用策略。
- **明确量化目标**（提升30%获客数）→ 设定成功标准，引导AI聚焦具体方向。
- **增加约束条件**（预算、团队规模）→ 让策略实际可行，避免天马行空。
- **指定输出格式与内容结构**（四部分，Markdown）→ 消除格式歧义，使结果符合用户预期。

### 7. 严重性评级

| 问题类型 | 严重性 | 说明 |
|--------|--------|------|
| 数据缺失 | **功能严重** | AI无法执行核心任务，输出必然不基于数据，导致策略无效或误导。 |
| 背景与目标模糊 | **功能中等** | 策略很可能泛泛而谈，实用性低下，需多次追问才能纠正。 |
| 输出格式未指定 | **功能轻微** | 用户可能得到不合适的形式，但内容本身尚可调整。 |
| 无安全风险 | 无 | 未涉及敏感或危险指令。 |

**总结**：该prompt存在功能性缺陷，尤其是缺失关键数据与上下文，导致AI无法产出有价值的定制策略。建议用户在提问前先准备好相关数据与目标，并明确期望输出格式。
