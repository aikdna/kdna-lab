### 最常见的系统性错误：任务混合

这是最普遍的问题。你很可能在一个 prompt 里塞了多个不同性质的任务，导致模型“精神分裂”。

**典型表现：**
- “请分析这份市场报告，总结三个要点，然后用它写一封给CEO的邮件，最后再输出一个JSON格式的行动计划。”
- “帮我写一篇关于气候变化的文章，同时检查它的语法错误，并把它翻译成英文。”

**为什么失败：**
模型在处理这种 prompt 时，会平均分配注意力。它可能把精力花在格式转换（JSON、翻译）上，而忽略了最核心的分析或创作任务。结果就是：**格式正确，但内容浅薄、逻辑混乱**。

**诊断方法：**
回顾你最近写的3-5个 prompt，数一数每个 prompt 里包含了几个**独立的、需要不同认知能力**的任务（比如：分析、总结、创作、格式化、翻译、批判）。如果超过1个，这就是问题所在。

**解决方案：**
**拆解。** 一个 prompt 只做一件事。把“分析报告”和“写邮件”拆成两个独立的对话或步骤。先让模型分析，得到结果后，再把分析结果作为上下文，让它写邮件。

### 第二常见的问题：目标模糊

你很可能知道“要什么”，但没想清楚“完美的输出能用来做什么”。

**典型表现：**
- “帮我优化一下这个商业计划书。”（优化什么？利润？风险？可读性？投资人吸引力？）
- “给我一些创意点子。”（关于什么？用于什么场景？成功的标准是什么？）

**为什么失败：**
模型无法读心。它只能根据字面意思猜测。如果目标不明确，它会输出一个“平均化”的、看起来合理但对你毫无用处的答案。这就是你感觉“太泛”、“太浅”的根本原因。

**诊断方法：**
试着用一句话回答这个问题：**“如果模型给出了完美的输出，我下一步能直接用它做什么？”**
- 如果答案是“直接发给老板”，那你的 prompt 需要包含老板的偏好和决策标准。
- 如果答案是“作为初稿，我再修改”，那你的 prompt 需要明确哪些部分必须准确，哪些可以自由发挥。
- 如果答不上来，说明你的目标本身就是模糊的。

**解决方案：**
**定义“成功”。** 在 prompt 里加入一句：“这个输出的成功标准是：它必须能直接用于说服投资人增加预算，因此需要包含ROI数据和竞品对比。”

### 第三常见的问题：上下文缺失

你给了模型一个任务，但没给它完成这个任务所需的“原材料”。

**典型表现：**
- “写一份法律合同。”（没给：管辖地、双方名称、交易标的。）
- “分析这个产品的定价策略。”（没给：成本结构、竞品价格、目标利润率。）

**为什么失败：**
你要求模型“分析”，但没给它数据。它只能基于自己的训练数据（可能过时或不相关）来“编造”分析。结果就是看似有道理，实则无根无据。

**诊断方法：**
检查你的 prompt：**模型要完成这个任务，需要哪些它不可能知道的信息？** 这些信息你提供了吗？比如：具体数据、内部流程、用户画像、历史背景。

**解决方案：**
**把上下文当作任务的一部分。** 在 prompt 里明确写出：“以下是完成此任务所需的数据：……”。不要假设模型知道你的具体情况。

---

### 总结：你的系统性问题很可能属于这三类之一

| 问题类型 | 核心症状 | 典型用户感受 | 首要行动 |
| :--- | :--- | :--- | :--- |
| **任务混合** | 输出格式完美但内容肤浅 | “感觉它什么都做了，但什么都没做好” | **拆解**为多个单任务 prompt |
| **目标模糊** | 输出看起来合理但无法直接使用 | “太泛了，不是我想要的” | **定义**输出的成功标准 |
| **上下文缺失** | 输出有逻辑但缺乏事实依据 | “它编造了一些东西” | **提供**完成任务所需的数据 |

**建议你做的第一件事：**
不要再去研究“怎么写更好的 prompt 结构”或“用什么角色”。先做一次**自我审计**：找出你最近最不满意的3个 prompt，用上面的三个维度去分析它们。大概率，你会发现它们都犯了同一个错误。先把这个错误改掉，效果会比任何格式优化都明显。
