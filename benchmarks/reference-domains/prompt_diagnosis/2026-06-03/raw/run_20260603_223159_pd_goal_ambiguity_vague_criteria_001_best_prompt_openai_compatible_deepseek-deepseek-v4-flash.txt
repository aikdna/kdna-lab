## 诊断报告：提示词“帮我写一份好的商业计划书。”

### 1. 用户实际意图与隐含假设
- **显式意图**：用户要求AI生成一份商业计划书，并期望其质量“好”。
- **隐含假设**：
  - AI知道什么是“好的”商业计划书（主观标准，因人而异）。
  - AI知道用户的行业、公司阶段、目标受众、产品/服务、财务数据等关键背景信息。
  - AI能自动判断格式、长度、语言风格等输出参数。

### 2. 任务模糊性
- **混合目标**：无明显混合，但“写一份计划书”与“好”是两个不同维度的要求，后者缺乏可衡量指标。
- **成功标准不明确**：用户未定义“好”的标准——是逻辑严谨？数据详实？符合投资人偏好？还是适合内部执行？导致AI可能产出泛化、空洞的内容。

### 3. 缺失的上下文（功能性关键缺失）
| 缺失信息 | 具体影响 |
|----------|----------|
| 公司/项目名称、行业、发展阶段 | 无法定制行业术语、竞争分析、增长策略 |
| 目标受众（投资人/银行/内部团队） | 决定语气、重点内容（如财务预测详略） |
| 计划书用途（融资/立项/申请贷款） | 影响结构侧重（如市场与财务权重） |
| 现有数据（收入、团队、产品状态） | 财务预测、运营计划无依据 |
| 希望的长度与交付格式（文档/演示） | 输出可能过长或过短，非用户预期 |

### 4. 输出格式期望
- **显式格式**：无。
- **隐含假设**：用户默认AI知道标准商业计划书结构（执行摘要、公司概览、市场分析等），但不同机构对结构要求不同。此外，无篇幅、语言、排版要求。

### 5. 风险措辞分析
- **“好的”**：过度模糊且主观。AI可能根据训练数据中的“常见好案例”生成，但未必符合用户场景。属于**功能性问题**，可能浪费用户时间。
- **“帮我写”**：权限过于开放。AI可能直接生成一份高度通用、缺乏针对性的计划书，用户需自行修改。
- 无否定从句或引导性问题，但“好”一词隐含了价值判断，可能让AI倾向于堆砌积极词汇而非提供实质分析。

### 6. 具体改写建议（修复所有缺陷）

**原始提示**：
> 帮我写一份好的商业计划书。

**修复后提示**（最佳实践模板，可让用户填空或直接使用假设场景）：
> 请为我撰写一份专业的商业计划书，用于__【融资 / 内部立项 / 申请贷款】__，面向__【投资人 / 银行 / 管理团队】__。
> 
> 背景信息：
> - 公司名称：__【例如：智途科技】__
> - 行业：__【例如：人工智能物流】__
> - 当前阶段：__【种子轮 / 成长早期】__
> - 核心产品：__【智能分拣机器人】__
> - 已有数据：__【年营收约500万，团队20人】__
> 
> 要求：
> - 结构：包含执行摘要、市场分析、产品与服务、商业模式、竞争分析、营销策略、管理团队、财务预测（3年）及风险分析。
> - 篇幅：正文8-12页，含图表。
> - 语气：专业、客观、数据驱动。
> - 格式：输出为中文Markdown，可导出为PDF。
> 
> 如果缺少关键信息，请先列出需要补充的清单，再进行撰写。

**修复针对的缺陷**：
1. **明确用途和受众** → 解决模糊性，让语气和重点有据可依。
2. **提供背景信息** → 补足缺失上下文，避免泛泛而谈。
3. **指定结构、篇幅、语气** → 固化输出格式期望，消除猜测。
4. **加入“缺信息先问”规则** → 防止AI自行假设错误数据，提升可靠性。

### 7. 严重性评级
| 缺陷类型 | 严重性 | 说明 |
|----------|--------|------|
| 任务模糊性（“好”的定义不清） | **功能性** | 可能导致核心内容偏离用户期望，需多次迭代 |
| 缺失上下文（行业、用途等） | **功能性 → 严重** | 无关键信息时，输出几乎不可用 |
| 输出格式未定义 | **功能性** | 用户可能需要额外解释或重新调整格式 |
| 开放权限（“帮我写”） | **功能性 → 中等** | 若AI生成错误数据（如财务预测），可能误导决策，但非安全风险 |
| 安全风险 | 无 | 未涉及敏感、危险或伦理问题 |

**总结**：该提示词**功能严重不足**，但无安全风险。通过结构化补充上下文和格式要求，可将原始一次性请求转化为高效、可定制的任务。
