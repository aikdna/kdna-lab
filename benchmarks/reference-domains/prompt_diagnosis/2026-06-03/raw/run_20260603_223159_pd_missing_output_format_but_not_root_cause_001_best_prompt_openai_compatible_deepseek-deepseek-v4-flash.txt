## 诊断报告：`"帮我分析竞争对手"`

### 1. 实际意图与隐含假设
- **显性请求**：要求AI进行“竞争对手分析”。
- **隐含假设**：假设AI知道用户所处的行业、具体的竞争对手、分析目标（如市场份额、产品对比、战略动向等）以及分析深度。实际上，这些信息完全缺失。

### 2. 任务歧义
该 prompt 混合了多种可能的分析目标（例如：SWOT分析、波特五力、产品对比、价格策略、市场占有率、用户评价等），但没有指定任何一项。**成功标准不明确**——AI无法判断什么样的回答才算“好”，容易给出泛泛而谈、缺乏针对性的内容。  
**严重等级**：功能性（中等偏高，因为输出大概率无用）

### 3. 缺失上下文
需要补充的关键信息包括：
- **行业/市场**（如：新能源汽车、SaaS、咖啡店）
- **具体竞争对手名称**（如：特斯拉、比亚迪；或只说“头部品牌”）
- **分析目的**（如：制定竞争策略、寻找差异化机会、评估威胁）
- **时间范围**（如：近半年的动态？历史对比？）
- **数据来源偏好**（如：财报、用户评论、新闻报道）
缺失这些背景，AI只能依赖训练数据中的通用知识，结果往往空洞。  
**严重等级**：功能性（高，若不补充则回答无效）

### 4. 输出格式期望
原始 prompt 只写“分析”，未指定任何格式。用户后来额外提问“输出格式是否应该用markdown表格？”——这表明用户自身也意识到格式不明确。  
**风险**：AI可能输出纯文本段落、列表、或随意结构，导致信息难以对比。若最终需要用于报告或展示，混乱的格式会降低可用性。  
**严重等级**：功能性（中等，可通过后续指令修正）

### 5. 风险措辞
- **过度宽泛**：`“分析竞争对手”` 属于“超泛指令”，给予AI几乎无限的自由度，可能产生全面但浅显的百科全书式回答，或者偏离用户潜在需求。
- **无否定/引导陷阱**：没有否定词或预设答案，主要风险是**信息过浅而非误导**（除非用户对“竞争对手”定义有特殊理解，但未说明）。  
**严重等级**：功能性（低中）

### 6. 具体重写建议

**原 prompt**  
```
帮我分析竞争对手
```

**改进版**  
```
请帮我分析【[公司名称]】在【[行业/市场]】中的主要竞争对手。  
分析维度包括：  
- 市场份额与增长趋势  
- 核心产品/服务差异化  
- 定价策略  
- 营销渠道与品牌定位  
- 近期重大动态（过去6个月）  

输出格式：使用Markdown表格列出每个竞争对手（最多5个），每列对应上述维度，并在表格下方补充简要结论（1~2段），以便为制定竞争策略提供参考。  
目的：了解当前竞争格局，找出我们的相对优势和劣势。
```

**各修改点说明**：
- **增加占位符 `[公司名称]` 和 `[行业]`** → 明确上下文，消除歧义
- **列出具体分析维度** → 将模糊的“分析”拆解为可衡量的小目标，降低任务歧义
- **指定时间范围** → 约束数据时效性，避免AI引用过时信息
- **明确输出格式（Markdown表格 + 结论段落）** → 满足用户对表格的潜在需求，同时通过结论确保深度
- **明确目的** → 帮助AI调整回答风格（战略导向，而非纯信息罗列）

### 7. 严重性总评

| 问题类型 | 具体表现 | 严重等级 |
|--------|--------|--------|
| 任务歧义 | 未定义分析目标与成功标准 | 🟡 功能性（高） |
| 缺失上下文 | 缺少行业、竞争对手、目的、时间、数据源 | 🟡 功能性（高） |
| 输出格式不明确 | 原始prompt无格式指定，用户事后追问 | 🟢 功能性（中） |
| 过度宽泛措辞 | “分析”一词自由度太大 | 🟢 功能性（中） |

**总建议**：在发送类似指令前，至少填写行业与具体竞争对手名称，并明确输出格式。本 prompt 若不修改，AI 的回答将大概率不符合用户预期，属于 **功能性缺陷**，而非安全风险。
