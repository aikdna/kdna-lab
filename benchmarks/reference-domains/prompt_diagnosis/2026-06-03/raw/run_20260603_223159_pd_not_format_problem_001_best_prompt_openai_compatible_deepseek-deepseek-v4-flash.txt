## 诊断报告：Prompt “分析市场趋势并给出建议”

### 1. 实际意图与隐含假设
- **显式意图**：要求AI执行两个任务——分析市场趋势 + 给出建议。
- **隐含假设**：用户假设AI知道“哪个市场”、“什么时间范围”、“建议给谁”、“建议的用途”等关键信息。实际上这些信息完全缺失，导致AI只能输出最泛化的内容。

### 2. 任务模糊性（功能性问题）
- **多目标混合**：分析 + 建议是两个不同层次的任务，但未明确二者的关系（例如：基于分析结果给出建议，还是分别独立输出？）。
- **成功标准不明确**：没有定义“好的分析”应包含哪些维度（数据、图表、原因、影响？），也没有定义“好的建议”应具备什么特征（可操作、有优先级、风险提示？）。

### 3. 缺失上下文（功能性问题）
- **市场领域**：股票、加密货币、房地产、消费品、技术？完全未指定。
- **时间范围**：过去10年、当前季度、未来预测？未说明。
- **目标受众**：投资者、企业高管、政府决策者、个人消费者？不同受众需要不同深度的分析和建议。
- **约束条件**：风险偏好、资金规模、行业法规、地域限制？缺失。
- **数据来源**：是否允许使用实时数据？是否需要引用权威报告？未说明。

### 4. 输出格式期望（功能性问题）
- **完全未指定**：AI可能输出一段文字、列表、报告、表格、甚至对话。用户没有给出任何格式约束，导致输出结构不可控，难以直接使用。

### 5. 风险措辞（轻微）
- **“给出建议”** 过于宽泛，可能诱导AI输出不切实际或过于笼统的建议（例如“建议投资”），缺乏可操作性。
- 无否定或引导性措辞，但“分析”一词本身是开放式的，容易导致AI选择最省力的路径（例如只罗列常识性趋势）。

### 6. 具体重写方案

**原Prompt**：
```
分析市场趋势并给出建议
```

**重写后Prompt**：
```
请分析2024年中国新能源汽车市场的关键趋势，包括但不限于：
- 销量增长数据（同比、环比）
- 主要政策变化（补贴退坡、双积分政策）
- 竞争格局（头部企业份额、新进入者）
- 技术演进（电池、智能驾驶）

基于以上分析，为一家中型电池供应商（年营收50-100亿元）提供三条具体、可操作的战略建议，每条建议需包含：
- 行动步骤
- 预期效果
- 潜在风险

输出格式：
1. 趋势分析：分点列出，每个趋势附带简要数据或事实支撑。
2. 战略建议：编号列表，每条建议包含“行动-效果-风险”三个子项。
3. 总字数控制在800字以内，语言简洁专业。
```

**每个修复对应的缺陷**：
- **指定市场领域**（新能源汽车）→ 解决“市场”缺失。
- **指定时间范围**（2024年）→ 解决时间模糊。
- **列出分析维度**（销量、政策、竞争、技术）→ 解决“分析”标准不明确。
- **明确受众**（中型电池供应商）→ 解决建议针对性问题。
- **要求具体数量**（三条建议）→ 避免泛泛而谈。
- **规定建议结构**（行动-效果-风险）→ 提升可操作性。
- **明确输出格式**（分点、编号、字数）→ 解决格式期望缺失。

### 7. 严重性评级

| 缺陷类型 | 严重性 | 说明 |
|----------|--------|------|
| 任务模糊性 | **功能性问题** | 导致输出可能完全偏离用户需求 |
| 缺失上下文 | **功能性问题** | 核心信息缺失，AI只能猜测 |
| 输出格式 | **功能性问题** | 输出结构不可控，难以复用 |
| 风险措辞 | **轻微** | 仅导致建议过于笼统，但不危险 |

**总体结论**：该Prompt的主要问题不是“格式或标点”，而是**信息密度严重不足**。即使使用完美标点，输出质量也不会改善。需要补充领域、时间、受众、分析维度、建议要求、输出结构等关键上下文。
