## 诊断结果

这个 prompt 的主要问题不是格式或任务混合，而是 **目标模糊** 和 **上下文严重缺失**。

### 1. 目标（outcome）完全未定义
- 用户只说了“写一段产品介绍”，但**没有说明产品是什么**（名称、品类、核心功能、目标用户、要传达的核心价值）。
- 完美输出应该能让用户做什么？是用于官网、电商详情页、社交媒体口号，还是销售手册？不同场景对内容的侧重完全不同。
- 当前所有约束（禁用词、长度、语气、语法）都是**否定性指令**，但没有一个正向的成功标准。这使得 LLM 只能猜测什么是对的，极易输出“安全但无实质”的内容。

### 2. 上下文（context）几乎为零
- 缺少**产品本身的信息**：你写什么产品？它解决了什么问题？用户是谁？相比竞品有何特点？
- 缺少**使用场景**：在哪里展示？是 B2B 技术白皮书风格，还是 C 端快消品文案？
- 缺少**目标受众偏好**：对“正式”和“随意”的定义完全取决于产品与受众。例如，一款儿童玩具的“不要太正式”与一款医疗器械的“不要太正式”天差地别。

### 3. 指令之间存在隐性冲突
- “不要太长，不要太短” → 没有范围，LLM 只能猜测标准。
- “不要太正式，不要太随意” → 对同一产品，这两个要求可能同时被满足吗？如果产品本身是专业工具，用户期望的“正式度”与“随意度”可能本就不矛盾，但 prompt 没有给出**中间状态的参照**。
- 禁用词列表虽然明确，但**没有说明允许使用的替代风格**（比如用“实用”“可靠”“简洁”等例子自然替代，而不是单纯禁止）。

### ⚠ 最危险的失败模式
即使 LLM 严格遵守了所有禁用词和语法规则，输出一段**格式正确、语气看似合理但空洞无物**的文字，用户也很可能觉得“还不错”但实际上完全无法用于业务——因为缺少产品核心信息，内容必然泛泛而谈。这正是 KDNA 强调的“正确格式+错误内容”的陷阱。

## 优化建议（按优先级排序）

1. **补全上下文**  
   至少提供：产品名称 / 一句话核心卖点 / 目标用户 / 展示渠道。  
   例如：“写一段产品介绍，产品是XX智能温控水杯，主打恒温6小时，面向职场白领，用于电商详情页。”

2. **明确正向目标**  
   用一句话描述“完美输出能让你做什么？”  
   例如：“这段介绍要能说服用户下单，重点突出省心和便携。”

3. **将模糊约束转化为可量化的边界**  
   - “不要太长，不要太短” → “控制在50~80字，适合短图文”。  
   - “不要太正式，不要太随意” → “像朋友推荐好物一样自然，但用词专业（避免网络流行语）”。  
   - 禁用词可以保留，但最好给出一个**允许使用的替代词列表**（如“可靠”“实用”“持久”）。

4. **检查是否真的需要一次完成**  
   当前 prompt 其实是一个任务（写产品介绍）+ 很多约束，不是任务混合。但如果后续还要修改风格、增加数据分析等，就应该拆成多个 prompt，而不是塞在一起。

---

### 一句话总结
**这个 prompt 的核心问题是“要什么”和“关于什么”都没说，却花大量精力约束“不能怎样”。先定义产品与目标，再修剪约束，否则 LLM 只能产出“安全但无用”的空洞文字。**
