这个 prompt 产出不好的核心原因是 **任务混合 + 目标模糊**，而不是缺少格式或角色设定。

### 具体问题

1. **四个独立任务揉在一起**  
   “写博客”、“做PPT”、“发社交媒体”、“回复评论” 是四个完全不同的输出类型，需要的上下文、结构和执行逻辑互不兼容。模型无法在一个回复里同时生成一篇完整的博客、一份可用的PPT内容、社交媒体文案和评论回复——它只能随机选择一个方向“糊弄”，或者给出极其浅层的草案。

2. **缺乏任何目标定义**  
   博客主题是什么？受众是谁？PPT的用途是演讲还是分享？发到哪个社交媒体？评论是关于什么内容的？完全没有提供。模型没有上下文，只能猜，导致输出大概率不切实际。

3. **隐含的“一次性完成”假设**  
   用户似乎希望一个 prompt 搞定所有环节，但现实中没有 LLM 能在一个回答里同时完成这些差异巨大的任务（如果需要高质量结果）。这属于认知上的任务打包，应该拆解。

### 建议的解决方向

- **拆分为 3-4 个独立 prompt**，每次只做一件事：  
  1. 先确定博客主题、核心观点、目标读者，然后写博客。  
  2. 根据博客内容生成 PPT 大纲或逐页内容。  
  3. 提取博客中的关键引语或摘要，写适配特定平台（如LinkedIn/Twitter）的帖子。  
  4. 提供几条模拟评论，让模型基于博客观点回复。

- **如果确实想在一个对话里完成**，需要明确优先级和输出顺序（例如：先输出博客全文，再输出PPT逐页文本，再输出三条社交媒体文案，再输出两条评论回复），但这样做依然会因长度和注意力分散而损失质量。

**一句话总结**：这不是格式问题，是任务和上下文的问题。请先明确你要的第一个产出是什么，然后单独给它足够的背景信息。
