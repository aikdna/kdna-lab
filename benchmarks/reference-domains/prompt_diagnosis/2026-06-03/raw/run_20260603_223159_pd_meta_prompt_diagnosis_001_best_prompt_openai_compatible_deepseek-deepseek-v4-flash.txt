### 诊断报告：关于“诊断我的提示写作”的元提示

#### 1. 识别实际意图与隐含假设

*   **明确意图：** 用户希望得到对其过往提示写作习惯的“系统性”问题诊断。关键词是“系统性问题”，这意味着用户不仅仅要表面建议，而是希望获得一个框架或模式层面的缺陷列表。
*   **隐含假设：**
    *   **假设1：** 用户确实有许多写过的prompt可以供分析。如果用户没有提供任何示例，AI无法进行“诊断”，只能给出通用模板。
    *   **假设2：** 在当前聊天中，用户认为AI已经或者能够“看”到其过往的无效提示。这在大模型对话中是错误的——这是一个全新的空会话或单轮对话，AI没有历史记录。
    *   **假设3：** “总是效果不好”是一个客观、可量化的描述，而非主观感受。AI无法判断用户的主观标准（例如，是结果不准确、格式不对，还是缺乏创意？）。

#### 2. 检测任务歧义

*   **严重混淆：** 这个提示混合了**两个完全不同的任务**：
    1.  **任务A（显性）：** 让我诊断“您（用户）的prompt写作方式”。
    2.  **任务B（隐性）：** 给我一个通用的“prompt写作常见缺陷清单”。
*   **问题：** AI必须做出选择。如果AI选择任务A，它会要求用户提供历史prompt，但这会立刻让提示失效（因为用户没给素材）。如果AI选择任务B，它给出的通用清单会显得很“空泛”，用户可能会觉得“这我早知道了”，导致进一步的不满。
*   **成功标准模糊：** 什么是“诊断成功”？是列出5个通用问题，还是必须提供针对性的“你个人”的复盘？用户没有定义。

#### 3. 缺失的关键上下文

*   **致命缺失：用户的示例prompt。** 没有具体的prompt文本，任何诊断都是空中楼阁。诊断需要证据。
*   **缺失：效果不佳的具体表现。** 是语法错误？输出格式不匹配？逻辑混乱？AI拒绝回答？还是逻辑正确但缺乏创意？不同的“效果不好”指向完全不同的病因。
*   **缺失：使用场景。** 用户是在写什么类型的prompt？（代码生成、创意写作、数据分析、角色扮演？）不同场景的典型缺陷不同。
*   **缺失：使用的AI模型。** 针对GPT-4、Claude 3、Gemini还是其他模型的提示策略不同。一个适合GPT-4的提示可能在Claude上效果很差。

#### 4. 输出格式期望

*   **完全未指定：** 用户没有说明希望以何种形式获得诊断。是：
    *   一段分析文本？
    *   结构化的列表（序号1, 2, 3...）？
    *   带有严重等级（如本文档）的表格？
    *   一段可操作的建议？
*   **隐含假设：** 用户可能默认为“一段有洞察力的文字”。但作为“专家诊断”，结构化的输出（如使用Markdown标题、列表、严重等级）会显著提升专业性和可读性。如果不指定，AI可能输出一段杂乱无章的段落。

#### 5. 风险措辞

*   **隐藏否定/挫败感：** “我写了很多prompt但总是效果不好”这句话带有强烈的负面情绪。AI可能倾向于过度同情或给出规避责任（blame-free）的回答，从而可能绕过真正的技术性问题。这也可能导致AI提供非常通用的安慰性建议，而非犀利的诊断。
*   **过度宽泛的权限：** “帮我诊断一下我的prompt写作方式” —— 这是一个开放到危险的程度。AI难以确定分析的范围（是分析语气？措辞？逻辑？还是架构？）。这会导致输出过于泛化，缺乏针对性。

### 严重等级评估

| 问题 | 严重等级 | 原因 |
| :--- | :--- | :--- |
| **1. 缺少具体prompt示例（致命上下文缺失）** | **功能级** | 没有药，无法开处方。任何诊断都将是盲猜，对用户毫无实际帮助。 |
| **2. 任务歧义（诊断vs. 通用清单）** | **功能级** | AI很可能产生与用户期望不符的输出，导致用户再次感到“效果不好”。 |
| **3. 输出格式未指定** | **外观级** | 虽然不影响核心信息，但会导致输出混乱，降低专业性和可操作性。 |
| **4. 隐含假设和情绪风险** | **功能级** | 可能导致AI给出错误的输出方向（过度共情或宽泛安慰），忽略实质性问题。 |

### 具体修复方案：Prompt重写

**原提示：**
> 我写了很多prompt但总是效果不好。帮我诊断一下我的prompt写作方式有什么系统性问题。

**重写后的诊断级提示：**

```markdown
# 角色
你是一位专业的提示工程诊断专家。你的任务是深入分析我提供的具体prompt样例，诊断其潜在的系统性缺陷，并给出可操作的修复方案。

# 任务
请严格分析下面我写过的3个prompt样本（请注意，这些都是我过去写过的、但效果未达预期的prompt）。基于这些样本，诊断我在提示写作中存在的系统性问题。

## 样本1：[请在此处粘贴你的prompt1]
## 样本2：[请在此处粘贴你的prompt2]
## 样本3：[请在此处粘贴你的prompt3]

# 输出格式要求
请按照以下结构输出你的诊断报告：

### 1. 每位样本的独立诊断
   - **样本编号：** (1/2/3)
   - **意图拆解：** 这个prompt实际在要求什么？
   - **主要缺陷：** (例如：指令冲突、缺失上下文、格式模糊等)
   - **可能后果：** 这个缺陷会导致AI输出什么结果？

### 2. 系统性缺陷总结
   - 找出跨越多个样本的共性问题（例如：总是使用否定句式、承诺不明确等）。

### 3. 修复优先级与建议
   - 按严重等级（安全>功能>外观）给出最需要优先修复的问题。
   - 对每个优先问题，提供一个具体的重写示例。

# 附加约束
- 不要提供通用的写作技巧，只基于我提供的样本进行分析。
- 请使用中文输出，语言需具体、专业、可操作。
```

### 修复说明

| 修复点 | 针对的缺陷 | 说明 |
| :--- | :--- | :--- |
| **设置明确角色** | 任务歧义, 隐含假设 | AI被明确指定为“诊断专家”，消除它可能误入“安慰模式”的风险。 |
| **强制要求提供样本** | 致命上下文缺失 | 在任务描述中直接划出 `[此处粘贴]`，迫使用户在提交前必须提供素材。这是最关键的一步。 |
| **定义输出结构** | 输出格式未指定 | 使用Markdown标题、列表和编号，确保输出高度结构化、易于阅读。用户可以直接看到“诊断-总结-建议”的逻辑流。 |
| **明确任务范围** | 过度宽泛权限 | 用“基于样本”“不要通用技巧”等约束，限制AI的自由发挥，确保其聚焦于具体材料。 |
| **增加“附加约束”** | 隐含假设、泛化风险 | 明确禁止AI输出通用知识，迫使其必须基于证据说话，提升诊断的质量。 |

**结论：** 您原本的提示本质上是一个**“元诊断”的初稿**。它包含了正确的方向（系统性分析），但严重缺少分析所需的**数据（样本）**和**工具（格式约束）**。按照上述重写版本执行，您将获得一份真正有洞察力的、对症下药的诊断报告，而不是一篇泛泛的“提示写作技巧”课程。
