## 诊断分析报告

### 原始提示
```
诊断："AI会取代人类工作吗？"
```

### 1. 实际要求 vs. 隐含假设
- **显式意图**：用户要求AI回答一个二元问题（会/不会取代人类工作）。
- **隐含假设**：用户可能期望AI给出有深度的分析、预测或辩论，而不仅仅是“是”或“否”。但提示词本身并未明确要求“分析”或“讨论”，导致AI只能自行猜测用户的深度需求。

### 2. 任务模糊性
- **多目标混杂**：该问题可被理解为：
  - 求事实答案（例如引用统计数据）
  - 求预测（未来趋势）
  - 求观点或伦理讨论
  - 求具体行业案例
  - 成功标准不清：用户没有定义什么是“正确”的回答，AI可能给出过于宽泛或过于简短的回复。

### 3. 缺失上下文
- 未定义“AI”的范围（弱AI、强AI、生成式AI？）
- 未定义“取代”的程度（完全替代 vs. 辅助 vs. 岗位消失 vs. 任务重组？）
- 未指定时间维度（短期、中期、长期）
- 未限定行业或工种（所有工作？还是特定行业？）
- 未说明用户角色（学生、职场人、研究者？）—— 不同角色需要不同深度的回答

### 4. 输出格式期望
- **完全未指定**：AI可能默认输出一段议论性文字，但用户可能希望得到列表、结构化的论点、利弊对比表等。缺乏格式约束会导致回答不可控。

### 5. 风险性措辞
- **过于宽泛**：允许AI自由发挥，可能产生以下风险：
  - 给出绝对的“会”或“不会”，导致误导（例如忽略复杂的社会经济因素）
  - 陷入“汽车取代马车”式的简单类比，缺乏严谨性
  - 可能引发无谓的焦虑或过度乐观
- **无否定陷阱**：问题本身是中立的，但“会取代”的措辞潜在地假设了“取代”是正在发生或必然发生的事件，可能引导AI偏向肯定回答。

### 6. 严重性评级
| 问题类型 | 严重性 | 说明 |
|---------|--------|------|
| 任务模糊性 | 🔴 **功能性** | 导致AI无法准确理解用户所需，输出可能偏离预期 |
| 缺失上下文 | 🔴 **功能性** | 缺少关键约束，回答可能过于空泛或片面 |
| 输出格式未指定 | 🟡 **结构性**（中度） | 虽然不影响正确性，但降低实用性和可读性 |
| 宽泛措辞 | 🟡 **轻微安全** | 可能产生片面结论，但不会直接引发有害行为 |

### 7. 具体重写方案

#### 修改后的提示（中文）：
```
请分析“AI会取代人类工作吗？”这个问题。要求如下：
- 设定时间范围：未来10年内。
- 聚焦于白领行业（如客服、翻译、数据分析、初级编程）。
- “取代”定义为：岗位数量减少50%以上，而非完全消失。
- 输出格式：先给出你的核心结论（一句话），然后通过三个“支持取代”的论点和三个“反对取代”的论点进行对比分析，最后给出一个平衡的预测和应对建议（100-150字）。
- 避免极端化表述，使用数据和文献（可引用如麦肯锡报告、世界经济论坛预测）。
```

#### 每项修正解决的问题：
| 修正点 | 解决的问题 |
|--------|------------|
| 设定时间范围 | 消除时间模糊性，使回答可验证 |
| 聚焦白领行业 | 避免讨论过于宽泛 |
| 明确定义“取代” | 消除语义歧义，统一度量标准 |
| 规定输出结构 | 确保回答组织清晰，对比充分 |
| 要求使用数据 | 提高回答的严谨性和可信度 |
| 避免极端化表述 | 降低误导风险，引导辩证思考 |

### 总结
原始提示过于简单，依赖AI的默认行为，容易产生模糊、片面或不符合用户预期的回答。通过添加时间、范围、定义、格式和证据要求，可以显著提升输出的实用性和可靠性。**建议在提出开放问题时，始终补充至少2-3个具体约束**（如时间、范围、输出样式）。
