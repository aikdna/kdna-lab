这个 prompt 的核心问题在于**目标模糊**和**上下文缺失**，而不是格式或指令不完整。

**诊断分析：**

1.  **目标模糊**：用户没有定义“总结”的标准。什么是“好的总结”？是列出三篇最重要的论文及其结论，还是对整体趋势进行综述，还是提取关键数据？用户无法在一句话内说清“完美输出能让我做什么”，说明 prompt 还没准备好。
2.  **上下文缺失**：缺少三个关键信息：
    - **领域**：什么领域的研究成果？AI、医学、材料科学？没有范围，模型会随意选择或输出空泛内容。
    - **时间范围**：“最新”是指过去一周、一个月，还是本年度？
    - **成果定义**：是指论文、报告、实验数据，还是行业应用？
3.  **没有任务混合**：这是一个单一任务（总结），所以不是任务混合的问题。

**改进方向（按优先顺序）：**

1.  **先明确结果定义**：问自己：“我拿到这个总结后，下一步要做什么？”例如：写文献综述引言、评估是否要深入阅读某篇论文、判断行业趋势等。
2.  **补充核心上下文**：添加领域、时间范围、成果类型和评估标准。
3.  **最后才考虑格式**：只有在结果定义和上下文清晰后，才需要指定格式（如列表、JSON、分点）。

**改进示例对比：**

- 原始：`“总结一下最新的研究成果。”`
- 改进：`“请总结过去3个月内，在自然语言处理领域关于‘大语言模型推理能力’的研究成果。列出3篇关键论文，每篇包括：核心方法、主要发现和局限性。格式为分点列表。我的下一步是评估是否需要将这些方法应用到我们的产品中。”`

这样，模型不再需要猜测，输出自然有针对性。
