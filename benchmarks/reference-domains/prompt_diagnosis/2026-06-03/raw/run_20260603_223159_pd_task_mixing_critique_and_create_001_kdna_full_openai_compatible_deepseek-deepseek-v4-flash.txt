这个 prompt 的问题根源在于**任务混合**和**上下文缺失**，而不是格式或指令不够详细。

1. **任务混合**：你在一句话里塞入了四个完全不同的任务——写报告、分析数据、写邮件、规划行程。LLM 在处理多任务时，注意力会分散，它既无法为每个任务分配足够的“思考预算”，也无法保持一致的输出质量。结果往往是每个任务都只给了一个非常浅的回应（比如气候变化报告变成两段百科摘抄，表格分析变成一句“看起来数据正常”，邮件写成模板套话，行程变成一个空泛的列表）。

2. **上下文缺失**：每个子任务都缺少最关键的信息：
   - 气候变化报告：面向什么受众？侧重科学、政策还是行动建议？长度和风格要求？
   - Excel 表格分析：表格内容是什么？分析目的是找趋势、异常值还是做预测？具体要看哪些指标？
   - 客户邮件：客户是谁？邮件目的是跟进、投诉还是介绍新品？之前有无沟通背景？
   - 行程规划：你的现有日程？优先级（工作/生活/紧急）？有无固定时间约束？

**改进方向**：把一个大 prompt 拆成四个独立的 prompt，每个只做一件事，并为每件事提供必要的前置信息。例如：

> 第一轮 prompt：  
> “请写一份面向投资人的气候变化报告，重点分析2023-2025年主要碳排放政策对新能源行业的影响，篇幅800字，结论需包含两个投资建议。”

> 第二轮 prompt：  
> “以下是一个Excel表格的CSV格式数据（附上数据），请帮我找出销售额连续三个月下降的省份，并列出每个省份下降幅度最大的产品类别。”

以此类推，每个 prompt 只有单一目标 + 足够的上下文，LLM 才能产出真正有用的结果。
