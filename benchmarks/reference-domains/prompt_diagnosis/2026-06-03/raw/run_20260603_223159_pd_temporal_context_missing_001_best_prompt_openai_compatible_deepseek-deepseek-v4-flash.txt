## 诊断报告：用户提示 “总结一下最新的研究成果。”

### 1. 识别真实诉求与隐含假设
- **显式意图**：要求人工智能对“最新的研究成果”进行总结。
- **隐含假设**：
  - 存在一组明确、公认且可获取的“最新研究成果”（未限定领域、来源）。
  - 人工智能具备实时获取最新信息的能力（实际依赖训练数据截止时间或联网插件）。
  - “最新”是绝对时间概念，而非相对于上下文。
  - 成果的“总结”方式（详细程度、篇幅、重点）由AI自行决定。

### 2. 任务歧义检测
- **多目标混合**：单一“总结”动作，但“最新”与“研究成果”均未定义边界，实际隐含了 **“筛选→归纳→概括”** 多个子任务，且无优先级。
- **成功标准模糊**：
  - 总结应覆盖多少项成果？
  - 每项成果需多详细？
  - 什么才算“最新”（时间窗口？）——如果AI无法获取实时数据，可能输出过期信息或编造“伪最新”内容。

### 3. 缺失上下文
- **领域/学科**：无指定（医学？AI？材料科学？），AI可能随机选取或给出泛泛之谈。
- **来源限制**：是学术论文、科技新闻、企业报告？不同来源侧重点差异大。
- **时间范围**：“最新”无明确量级（上周？上月？近一年？）。
- **目标受众**：用户是学生、研究员还是高管？影响术语深度和结构。
- **语言/格式偏好**：中文口语、学术摘要、要点列表？

### 4. 输出格式期望分析
- **完全未指定**：AI可能输出一段文字，也可能输出带标题的列表、或引用来源。缺少格式约束导致结果不可控。

### 5. 风险措辞识别
- **严重性：功能性**
  - **“最新”的模糊性**：若AI没有实时数据，会倾向于使用训练数据中最新（可能已过时）的信息，甚至编造“最新”细节。
  - **“研究成果”过度宽泛**：缺乏约束可能使AI输出综合性但无实质内容的套话，或者随机聚焦无关领域。
  - **未提及引用**：用户未要求来源，AI可能不提供出处，降低可信度。
