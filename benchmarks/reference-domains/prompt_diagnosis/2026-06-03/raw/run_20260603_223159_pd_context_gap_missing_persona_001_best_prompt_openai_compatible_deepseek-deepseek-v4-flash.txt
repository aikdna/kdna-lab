## 诊断报告：Prompt「写一封邮件给团队」

### 1. 实际请求与隐含假设
- **显式请求**：生成一封收件人为“团队”的电子邮件。
- **隐含假设**：用户默认 AI 知道邮件的目的、语气、内容结构、发送人身份等。但这些信息完全缺失，导致 AI 只能随机生成一个通用模板，很可能与用户实际意图不匹配。

### 2. 任务模糊性（严重程度：功能性）
- **混合目标**：没有明确目标。邮件可以是通知、感谢、批评、任务分派、活动邀请等多种类型，每个类型需要完全不同的内容和语气。
- **成功标准**：用户没有定义“符合预期”的具体标准，AI 无法判断输出是否正确。例如，用户可能期望一封正式的项目进度通报，但 AI 可能生成一封非正式的团建通知。

### 3. 缺失上下文（严重程度：功能性）
需要补充的关键信息包括：
- 邮件主题 / 目的（如：团队周报、项目更新、会议邀请、提醒截止日期等）
- 发送者身份与团队关系（老板/同事/外部顾问？）
- 团队角色与文化（正式 vs 轻松？)
- 具体内容要点（需要包含哪些信息？）
- 是否有附件、截止日期、行动要求等
- 邮件语言的风格（中文商务、英文、中英混杂？）

### 4. 输出格式期望（严重程度：美观性/功能性）
- 用户期望 AI 输出一封邮件，但未指定格式细节（主题行、称呼、正文分段、签名等）。AI 可能会输出纯文本或带简单标头的邮件，而用户可能期望更结构化的格式（如正式商务邮件）。  
- 此外，是生成一封完整的邮件还是一段大纲？未说明。

### 5. 风险措辞分析（无重大风险）
- 没有否定措辞或诱导性问题，但“写一封邮件”属于**过度宽泛的权限**，导致 AI 可以自由发挥，容易偏离用户真实需求。
- 无安全或伦理风险。

### 6. 具体重写建议

#### 原始 Prompt（有缺陷）：
```
写一封邮件给团队
```

#### 改进版 Prompts（评分：清晰、可执行）：

**方案 A（任务明确版）**  
```
写一封邮件给团队，主题为“本周项目进度更新”。  
要求：
- 语气正式，但保持鼓励
- 内容包含：已完成任务、待处理事项、下一阶段目标
- 结尾敦促团队成员在周五前回复各自的进度
- 使用中文，邮件格式包括主题行、称呼、正文、结束语和签名
```

**方案 B（快速模板版）**  
```
请生成一封中文商务邮件范本，收件人为“技术团队”，发件人为项目经理。  
邮件目的：通知团队下周一（10月30日）下午2点举行紧急会议，讨论服务器迁移方案。  
要求：语气专业但紧急，注明需要回复是否参会，并在邮件末尾附上“如有疑问请联系我”字样。
```

**方案 C（多选预设版，适合实验）**  
```
请为以下三种场景分别生成一封发给团队的邮件，每种用<EMAIL>标签分隔：
1. 庆祝团队完成Q2目标
2. 提醒明天下午的代码评审会议
3. 通报客户投诉并要求团队反思

每封邮件需包含主题、称呼、正文和落款，语气与场景匹配。
```

### 7. 严重性总结

### 最终诊断结论
原始 prompt 的核心问题在于**信息缺失**——用户没有提供任何约束或背景，导致 AI 只能猜测。输出“不符合预期”的根本原因是用户自己的意图未被编码到 prompt 中。修复方法是：明确邮件的目的、受众、语气、内容要点及格式要求。
