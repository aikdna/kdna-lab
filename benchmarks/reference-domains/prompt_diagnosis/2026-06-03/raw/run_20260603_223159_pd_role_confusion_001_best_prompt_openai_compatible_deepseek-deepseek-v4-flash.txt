## 提示诊断报告

### 原始提示
```
诊断："作为一个医生，帮我分析这个投资组合的风险。"
```

### 1. 实际请求与隐含假设
- **显式请求**：要求AI扮演“医生”角色，对“投资组合”进行“风险分析”。
- **隐含假设**：认为医生角色能够泛化到金融领域，或期待AI用医疗诊断的框架来评估金融风险。这种跨界假设缺乏合理性——医生没有金融风险分析的专长，强行类比可能产生误导。

### 2. 任务模糊性
- **混合目标**：将“医生”角色与“投资风险分析”嫁接，导致目标混乱。用户究竟想要医疗健康风险评估（如投资压力对健康的影响），还是传统的金融风险管理？指令未明确。
- **成功标准缺失**：没有定义“风险”的具体维度（市场风险、信用风险、波动率等），也没有说明分析深度（定性描述、定量计算、建议等）。

### 3. 缺失上下文
- **投资组合信息**：未提供任何持仓、资产类别、比例、历史表现或目标（如退休、短期投机）。
- **背景约束**：缺少用户的风险承受能力、投资期限、市场环境等关键参数。
- **角色背景**：为什么选择“医生”角色？用户可能误以为“诊断”适用于任何分析场景，但未解释医生的具体知识领域（如金融心理、健康投资？）。

### 4. 输出格式期望
- **未指定**：输出格式完全隐含，AI可能自由生成段落、列表或类比。缺乏结构化要求（如先列出风险类型，再给评分，最后建议），不利于结果一致性。

### 5. 风险措辞
- **角色误导**：“作为一个医生”可能导致AI生成医学类比（如“投资组合的‘病情’是过度集中于单一资产”），而非专业金融分析。这降低可信度，甚至造成用户误判。
- **过度宽泛**：“分析……风险”未限定范围，AI可能泛泛而谈（如“所有投资都有风险”），无法满足用户真实需求。
- **否定陷阱**：无直接否定，但角色扮演可能引发AI回避金融术语（因其“医生”身份），导致输出偏离实际。

### 6. 具体重写方案

**重写后的提示**：
```
你是一位金融投资顾问。请分析以下投资组合的风险：

【投资组合信息】
- 资产配置：60%股票（其中50%科技股），30%债券（国债），10%现金
- 总投资金额：100万元
- 投资期限：5年
- 风险承受能力：中等

请按以下格式输出：
1. 主要风险类型（市场风险、集中风险、流动性风险等）
2. 针对该组合的风险等级（低/中/高）
3. 具体风险点（如科技股占比过高）
4. 优化建议（如有）
```

**修改说明**：
| 修复项 | 原问题 | 修改后 |
|--------|--------|--------|
| 角色匹配 | 医生→金融不相关 | 改为“金融投资顾问”，直接匹配任务 |
| 上下文缺失 | 无组合信息 | 提供资产构成、期限、风险承受能力（可替换为真实数据） |
| 输出格式 | 未指定 | 明确结构化要求（四项清单） |
| 任务模糊 | 风险定义不清 | 列出需评估的风险类型，并给出示例 |
| 安全/功能性 | 角色误导可能产生错误类比 | 消除角色混淆，聚焦金融专业领域 |

### 7. 严重性评级
| 缺陷 | 等级 | 说明 |
|------|------|------|
| 角色不匹配 | **功能性** | 直接导致分析方向错误，输出实用性差 |
| 上下文缺失 | **功能性** | 若无补充信息，AI只能生成通用答案，用户无法获得定制化建议 |
| 输出格式未指定 | ** cosmetic** | 输出虽可用但可能混乱，降低可读性，不影响核心正确性 |
| 角色误导（潜在安全） | **安全-关键（低概率）** | 若用户误信“医生”身份下的金融建议可能造成经济损失，但实际风险低于医疗误诊 |

**总结**：该提示的核心缺陷是**角色与任务不匹配**，以及**关键上下文缺失**。重写后应明确专业领域、提供具体数据、规定输出格式，以消除模糊性。
