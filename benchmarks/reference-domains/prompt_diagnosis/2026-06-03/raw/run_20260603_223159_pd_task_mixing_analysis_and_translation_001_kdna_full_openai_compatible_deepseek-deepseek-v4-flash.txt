这个 prompt 存在三个核心问题：

1. **任务混合**：它同时要求“分析性能瓶颈”、“翻译成 Python”、“写 README”。这三个任务目标不同、输出格式不同、需要的上下文也不同。模型会分散注意力，导致每个任务都做得浅——比如分析可能只是泛泛而谈，翻译可能忽略原逻辑细节，README 可能缺乏针对性。

2. **目标模糊**：没有定义“性能瓶颈”的具体标准（是时间复杂度、内存、I/O 还是其他？），也没有说明“翻译成 Python”时是否需要保持原算法结构、优化或适配 Python 特性。README 的受众、详细程度、包含哪些内容（安装、示例、API 说明？）都不明确。

3. **上下文缺失**：没有提供待分析的代码本身，也没有说明代码的运行环境、输入规模、使用场景。模型无法进行任何实质性的分析或翻译。

**改进方向**：
- 先拆分为三个独立的 prompt，每次只做一件事。
- 每个 prompt 都要明确**输出目标**（例如：分析后要给出什么结论？翻译后要能直接运行？README 要帮助谁快速上手？）。
- 提供必要的上下文（代码、环境、性能关注点等）。

例如，第一个 prompt 可以改为：  
“请分析以下代码的性能瓶颈，重点关注时间复杂度，并指出最耗时的部分。代码：<粘贴>”  
完成后再进行下一步。
