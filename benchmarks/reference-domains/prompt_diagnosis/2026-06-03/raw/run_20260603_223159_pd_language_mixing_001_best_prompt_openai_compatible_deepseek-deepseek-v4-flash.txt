## 诊断报告

### 原始提示词
```
Write a blog post about machine learning 然后用中文解释给同事 最后翻译成日文给客户
```

---

### 1. 实际要求与隐含假设

- **显式要求**：完成三个连续动作——①用英文写一篇关于机器学习的博客文章；②用中文向同事解释该内容；③将内容翻译成日文给客户。
- **隐含假设**：  
  - 博客文章默认用英文撰写（因为“Write a blog post”是英文指令）。  
  - “解释”和“翻译”的对象是同一篇博客内容。  
  - 输出应包含三个独立部分，且顺序固定。  
  - 同事和客户对机器学习的了解程度、需求背景相同（未区分）。

### 2. 任务模糊性（功能性问题）

- **多目标混杂**：一个提示中混合了“创作”、“解释”和“翻译”三个不同性质的任务，且未说明是否要一次性输出全部结果，还是分步执行。  
- **成功标准不明确**：  
  - 博客的篇幅、风格（技术性/科普性）、目标读者？  
  - “解释”的深度（面向非技术同事？还是技术同事？）  
  - “翻译”的正式程度（商务邮件？技术文档？）  
- **语言关系混乱**：博客用英文写，解释用中文，翻译成日文——但解释和翻译是否基于同一英文原文？如果是，则解释和翻译应分别输出；如果不是，则逻辑矛盾。

### 3. 缺失上下文（功能性/安全性问题）

- **主题范围**：“machine learning”过于宽泛，未指定具体子领域（如监督学习、深度学习、应用案例等）。  
- **受众信息**：  
  - 同事：是否懂机器学习？需要基础解释还是高级讨论？  
  - 客户：日本客户是技术团队还是业务方？是否需要文化适配？  
- **输出格式**：未说明最终交付物是三个独立段落、一个分节文档，还是仅需最终日文版本。  
- **约束条件**：无字数限制、无风格要求、无术语处理规则（如是否保留英文术语）。

### 4. 输出格式期望（功能性问题）

- **完全未指定**：AI 可能输出一段混合文本（如先写英文博客，然后中文解释，最后日文翻译），也可能只输出日文翻译，或仅输出中文解释。用户期望不明确，导致结果不可控。

### 5. 风险措辞（轻微）

- **“然后”和“最后”**：暗示顺序，但未说明是否要保留中间产物。若 AI 只输出最终日文，则丢失了前两步。  
- **过度宽泛**：没有限制 AI 的创作自由，可能导致内容偏离实际需求（例如写一篇过于学术的博客，而同事需要通俗解释）。

### 6. 严重性评级

| 问题类型 | 严重性 | 说明 |
