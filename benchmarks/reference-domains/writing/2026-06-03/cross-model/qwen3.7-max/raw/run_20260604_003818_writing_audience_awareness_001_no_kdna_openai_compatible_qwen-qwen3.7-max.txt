### 1. 结构化与一目了然 (Structure & Scannability)
开发者通常是“带着问题来查文档”的，我会帮您把大段文字拆解为清晰的模块：
*   **端点 (Endpoint)**：HTTP 方法 + URL 路径。
*   **一句话描述**：这个 API 解决什么具体问题。
*   **认证与限流**：需要什么 Header？每分钟能调多少次？

### 2. 参数与响应的“契约”清晰 (Clear Contracts)
*   **请求参数**：使用表格明确区分 Path、Query、Body 参数，并标明**数据类型**、**是否必填**、**默认值**和**枚举值范围**。
*   **响应数据**：不仅提供 JSON 示例，还会对关键字段进行注释说明。

### 3. 开箱即用的代码示例 (Copy-Paste Ready)
*   提供 `cURL` 命令（最基础）。
*   根据您的需求，补充主流语言（如 Python, Node.js, Java, Go）的代码片段。
*   提供**成功**和**失败**两种真实的 JSON 响应示例。

### 4. 友好的错误处理 (Developer-Friendly Errors)
*   将内部错误码转化为开发者能看懂的“排查指南”。
*   说明 HTTP 状态码（如 400, 401, 429, 500）在当前 API 中的具体含义。

### 5. 语言风格转换 (Tone & Language)
*   把“内部黑话/系统术语”翻译成“通用开发者语言”。
*   使用主动语态（例如：“使用此接口获取...” 而不是 “此接口被用于获取...”）。
